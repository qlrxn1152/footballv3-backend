# 회원가입

## 기능 목적 

- 사용자가 아이디와 비밀번호를 통해, 해당 프로젝트에 회원이 될 수 있다.

## 회원 정보 

- 아이디
- 암호화된 비밀번호 ( 해쉬 )
- 회원 레이팅
- 누적 득점 
- 회원 권한
- 가입 일시

## 기본값 

- 회원 레이팅 : 1500 
- 누적 득점 : 0 
- 회원 권한 : USER 

## 비즈니스 규칙 

- 아이디와, 비밀번호는 비어 있을 수 없다. 
- 아이디는 최소 4글자, 최대 12글자까지만 허용한다.
- 유저가 입력하는 비밀번호는 최소 4글자, 최대 15글자까지만 허용한다.
- 아이디는 중복일 수 없다.
- 회원가입 전용 요청 / 응답 DTO 가 존재한다.
- 비밀번호는 평문으로 저장하지않고, 암호화해서 저장한다.
- 회원가입시, 권한은 자동으로 USER 로 지정된다.
- 회원가입할때 유저가 입력하는값은 아이디 / 비밀번호 뿐이다.

## 구현 순서

- Member 엔티티, MemberRole Enum
- MemberRepository
- 비밀번호 암호화 
- 회원가입 서비스 
- 회원가입 API ( controller )
- 예외 처리
- 테스트 검증 , Swagger 검증
